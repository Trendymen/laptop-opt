# 蛟龙 16 Pro 性能调校交接页设计

日期：2026-07-12
状态：视觉与内容方案已确认，待书面评审

## 1. 目标

制作一份交给这台电脑下一任使用者的设备交接页面，说明：

- 电脑当前已经完成的硬件、BIOS、系统与软件调整；
- 参数回退、软件启动异常或设置被重置时，如何查图复现；
- 之后可以尝试的调整，以及执行前提和可接受温度范围；
- 已安装的驱动、监控与测试工具分别用于什么场景。

页面是可操作的设备交接手册，不是面向其他机型的公开教程。所有数值都标注为“这台设备当前记录值”，不表达为适用于其他设备的通用参数。

## 2. 读者与成功标准

主要读者是这台电脑之后的使用者。读者不需要预先了解 CCD、PBO、UMAF、内存时序或压力测试。

成功标准：

1. 只看首屏与“当前交接基线”，即可判断关键配置是否回退。
2. 每项调整都提供与其适用的信息；用户已经提供截图或异常路径的项目，必须展示对应证据图或恢复步骤。
3. 页面可在常见桌面宽度下生成连贯的浏览器全页截图。
4. 最终只需移动一个 HTML 文件，断网仍能查看全部正文和图片。
5. 只有打开 B 站教程时需要联网。

## 3. 最终交付物

主交付物为一个独立 HTML 文件，例如 `laptop-performance-handoff.html`。

- HTML、CSS、JavaScript、SVG 图标全部内联。
- 10 张用户图片全部转换为 WebP，再以 `data:image/webp;base64,...` 形式内联。转换保持原始像素尺寸，以无损或近乎无损为质量基线。
- 不依赖外部字体、CDN、图片目录或本地绝对路径。
- B 站教程保留为普通外部链接，并在页面中显示完整标题、BV 号、用途和关键时间点。
- 源图片与生成脚本可以留在工程目录中便于重建，但最终 HTML 单独复制到空目录后必须完整可用。

以上内容只属于工程实现与验收要求，不得作为正文呈现给下一任使用者。渲染后的页面不显示“单文件 HTML”“Base64”“Data URI”“图片内联”“离线能力”“交付格式”等前端实现细节。

## 4. 视觉与交互方向

### 4.1 视觉论点

采用已确认的 A 方案“机能交接日志”：哑光黑色底、低饱和紫色结构线、酸性荧光绿参数强调，结合大面积真实设置截图，形成克制、可信、带设备工程感的长页。

页面不使用常规 SaaS 卡片墙。信息通过章节编号、分隔线、跨栏媒体、参数排版和留白建立层级。

页面根画布铺满浏览器可用宽度，不使用居中的固定最大宽度外框。滚动由浏览器文档本身承担，不创建内部滚动容器。关键参数不低于 28px，正文不低于 16px，辅助说明不低于 14px，纯元信息标签不低于 12px，保证宽屏截图缩放后仍能阅读。普通文本对背景的对比度不低于 4.5:1，大号文字不低于 3:1。

### 4.2 内容计划

- Hero：设备身份、交接目的、产品图和四个关键基线。
- Support：不可删除 UMAF 分区等最高优先级警告。
- Detail：CPU、显卡与电源、UXTU、内存与 UMAF、未来调整、软件工具。
- Final：一次只改一项、先截图再测试、不稳定时回到基线。

页面不设置“交付格式、图片资源、离线查看、外部链接数量”等面向开发者的说明区。教程只在对应操作章节中自然出现。

### 4.3 交互论点

页面以全页截图稳定性优先，只保留三类克制交互：

1. 首屏文字与主图的短促进入动画；
2. 章节进入视口时的轻微上移和淡入；
3. 图片悬停提示与点击大图查看。

所有动效遵守 `prefers-reduced-motion`。页面支持仅供截图与验收使用的 `?capture=1` 查询参数：脚本在首帧前为根元素设置 `capture-mode`，关闭动画、把所有 reveal 状态设为最终可见、避免固定定位并隐藏无关悬浮控件。该实现细节不在正文中显示。

用荧光绿横条明确说明：

- 约 80MB 的磁盘分区是 UMAF，不可删除或格式化；
- 不确定参数含义时，不要一次修改多项；
- BIOS、负压、超频调整可能造成不稳定，修改前保留当前值截图。

### 5.3 章节 00：当前交接基线

以最快扫描方式列出：

- CCD1 已关闭；
- CPU 睿频已关闭；
- Windows 系统电源计划中已调出“高性能”计划，并用于设置处理器最大频率；
- 机械革命控制台中的“高性能电源模式”保持关闭；

### 5.5 章节 02：显卡与电源模式

- 机械革命控制台保持独显直连；
- 机械革命控制台保持关闭“高性能电源模式”；
- 两张设置截图按“独显直连 → 高性能电源模式关闭”的顺序上下竖直排列，均按原始宽高比完整展示，文案直接点明目标选项。

### 5.6 章节 03：UXTU 负压与启动异常

- 当前设置：AMD Curve Optimiser 的 All Core Offset 为 -20；
- -20 是本机记录值，不是通用推荐值；
- 如果出现蓝屏、游戏崩溃、重启或其他不稳定，优先恢复默认并重新验证；
- 如果 UXTU 没有开机自启且手动打开无反应，删除 `AppData\Local\JamesCJ60` 下截图红圈所示的 Universal x86 Tuning Utility 配置文件夹；
- 明确提醒不要误删旁边的压缩包。

### 5.7 章节 04：双通道内存与 UMAF

这一章必须先强调“当前稳定运行结果”，再说明恢复入口：

1. 原装 32GB 单条已替换为两条 16GB，组成 32GB 双通道；
2. ZenTimings 截图代表当前 Windows 中已经稳定运行的实际结果；
3. 5600 MT/s 已经是这台电脑当前调整完成并稳定运行的频率，正常情况下保持不动，不继续提高或重新调整；
4. UMAF 中的 DDR SPD Timing 与 DDR Non-SPD Timing 截图只是产生当前结果的参数对照，不作为未来继续超频的建议；
5. 如果 BIOS/UMAF 设置被回退，只对照两张 UMAF 截图中明确出现的字段恢复；这些图片仅覆盖部分时序与小参数，不能承诺完整恢复全部 BIOS/内存设置；
6. 如果内存出现稳定性问题并需要更保守的设置，可从截图中已记录的字段出发，逐项放宽或回退参数，再重新验证；未记录的参数不得凭空推断；
7. UMAF 进入方式：开机或重启开始时持续按 F2，选择最右边第三项，再进入后续界面的第二项；
8. BIOS 解锁、UMAF 安装与进入方法再次指向教程 01。

页面用明确文案区分：`ZenTimings = 当前 Windows 实际运行结果`，`UMAF = 已记录字段的局部参数入口和回退对照`。ZenTimings 图片作为本章主证据放大展示，并带“当前稳定运行结果”状态标识。两张 UMAF 图片放在次级的“局部回退 / 恢复参考”区域。

内存操作原则必须直接写明：`5600 MT/s 正常保持；只有发生稳定性问题时，才考虑降频、放宽或回退已记录参数，并逐项验证。`

两张 UMAF 图片已由用户重新裁剪。页面按“DDR SPD Timing → DDR Non-SPD Timing”的顺序上下竖直排列，并以各自原始宽高比完整展示，不再做 CSS 视窗裁切；点击后仍查看同一份完整内联 WebP。

### 5.8 章节 05：之后可以怎么调

- 使用上风压散热器之后，再尝试把 Windows 高性能电源计划中“接通电源”的最大处理器频率从 5000 MHz 调到 5200 MHz；电池侧 4500 MHz 不随之修改；
- 5200 MHz 必须标注为“后续可尝试”，不能写成当前设置；
- 本章的后续调整只针对 CPU 定频和散热条件；内存 5600 MT/s 不属于待提高项目；
- 最终游戏温度稳定在 85–87°C 属于用户给出的可接受范围；
- 一次只改变一个变量，保留截图并进行实际游戏与稳定性验证；
- 出现异常时，先回到本文记录的当前基线，再逐项定位。

## 7. 技术结构

不使用 React 或运行时框架。页面是静态长页，采用：

- 语义化 HTML；
- 原生 CSS，包括响应式、打印和减少动效样式；
- 少量原生 JavaScript，用于大图查看和进入动画；
- 内联 SVG 图标；
- Node ESM 构建脚本，按上述质量规则转换 WebP、执行图像质量校验、Base64 编码并写出最终 HTML。

构建脚本遇到缺失图片、转换失败或空数据时必须以非零状态退出，并指出具体文件。最终 HTML 不引用工程内相对图片路径。

## 8. 全页截图约束

- 以 1440px 与 1920px 桌面宽度作为主要验收视口，同时保证窄屏可读；
- 页面根画布铺满浏览器可用宽度，滚动元素必须是文档本身，不允许章节或外框形成内部纵向滚动区；
- 宽屏下关键参数不低于 28px，正文不低于 16px，辅助说明不低于 14px，纯元信息标签不低于 12px，不以过小字号换取密度；
- 正文图片直接出现，不使用轮播、手风琴或必须点击后才能显示的内容；
- 图片不采用懒加载，避免截图时未加载；
- 不使用会在全页截图中重复出现的固定头部或固定侧栏；
- 章节间保持足够留白，但避免纯装饰造成页面过长；
- 通过 `?capture=1` 进入截图模式，关闭动画、滤镜和不必要阴影，并保证所有内容在首帧处于最终可见状态。

## 9. 验证与验收

### 9.1 构建检查

- 构建命令成功退出；
- 最终 HTML 存在且大小非零；
- HTML 中包含全部 10 个源图素材的内联数据；
- 10 个图片 Data URI 的 MIME 类型必须全部为 `image/webp`，且解码后尺寸与对应源图一致；
- 9 张证据图转换前后的解码 RGBA 像素必须完全一致；首屏产品图必须达到 SSIM ≥ 0.999、PSNR ≥ 50 dB；
- 不存在 `src="./..."`、`src="assets/..."`、外部 CSS、外部 JavaScript 或外部字体；
- 外部 HTTP(S) 链接只允许教程 `<a href>` 指向 B 站。
- 渲染后的可见文本不得包含“单文件 HTML”“Base64”“Data URI”“图片内联”“离线查看”“交付格式”等实现说明。

### 9.2 离线检查

把最终 HTML 单独复制到空目录并断网打开：

- 所有文字、图片和版式完整；
- 大图查看仍可用；
- 教程标题、BV 号、用途与 URL 仍可阅读；
- 点击教程时允许因断网无法访问，但页面本身不受影响。

### 9.3 浏览器与视觉检查

- 在 1440px 桌面视口生成全页截图；
- 检查没有图片裁断、横向溢出、文字遮挡或动画停在中间状态；
- 检查 10 个素材均有正文可见预览；显卡/电源两图与两张 UMAF 图分别上下排列，均按原始比例完整展示并可点击查看；
- 检查移动端布局仍可阅读，首屏产品图完整显示全部宽度且不被横向裁切；
- 检查链接键盘可聚焦、图片有替代文本；普通文本对比度至少 4.5:1，大号文字至少 3:1；
- 检查 `prefers-reduced-motion` 下无强制动画。
- 检查 1440px 与 1920px 下均为页面级滚动，没有居中窄画布造成的外侧滚动条错觉；
- 检查 ZenTimings 被明确标为当前稳定结果，5600 MT/s 被明确标为正常情况下保持不动，UMAF 被明确标为回退 / 恢复参考。

## 10. 明确不做

- 不做在线后台、表单或参数自动保存；
- 不自动检测电脑当前 BIOS、功耗或内存状态；
- 不把本机参数包装为其他电脑可以直接照抄的通用建议；
- 不嵌入 B 站视频播放器，避免破坏单文件离线能力和全页截图稳定性；
- 不修改用户桌面上的原始图片。
- 不向下一任使用者展示构建方式、文件格式、Base64、图片内联、离线实现或其他前端技术细节。
- 不对包含界面文字、参数或 BIOS 内容的证据图进行有损压缩；不以减小最终 HTML 体积为由降低图片质量。
