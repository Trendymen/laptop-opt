# 页面内容优先级重排设计

日期：2026-07-13
状态：设计已确认，待书面评审

## 1. 目标

把页面从“按调校过程顺序展示的长教程”调整为“下一任使用者先看结论和下一步，再查证据”的设备参考页。

首屏必须明确说明这台电脑已经完成哪些关键调整，包括内存超频和时序调整。教程、操作截图和恢复证据保留，但后置展示。

## 2. 当前问题

- Hero 已显示部分参数，紧接着的“00 当前设置基线”再次总结相近内容，形成连续重复。
- “之后可以怎么调”和“已经安装的软件”是接手者最先需要的信息，却位于 05、06。
- “内存 5600 MT/s”和“16GB × 2”虽然出现在首屏，但没有明确表达“内存超频和时序已经调整完成”。
- 教程和大尺寸设置截图在行动信息之前占据大量滚动长度。

## 3. 首屏设计

删除独立的“00 当前设置基线”章节，把唯一一份当前状态摘要整合进 Hero。

Hero 下方使用一组“已完成调整”信息，明确表达：

1. CCD1 与 CPU 睿频已关闭，接电频率为 5.0GHz；
2. UXTU 全核负压为 -20；
3. 显卡保持独显直连；
4. 内存已更换为 16GB × 2 双通道；
5. 内存已经超频并稳定运行在 5600 MT/s；
6. 内存时序已经调整并稳定运行。

首屏不得只给出 `5600 MT/s` 数字而省略“内存超频、时序调整”的含义。

原 UMAF 专属警告不再占据首屏。首屏只保留通用操作原则：一次只改一项、修改前截图、不稳定时回到当前记录值。

## 4. 新的信息顺序

### 01：之后可以怎么调

把原 05 提升为 01，首先回答接手者“下一步能做什么”：

- 使用上风压散热器后，再考虑把接电频率从 5000MHz 调到 5200MHz；
- 电池侧 4500MHz 不改；
- 游戏温度 85–87°C 是用户提供的可接受范围；
- 一次只改一个变量并验证。

内存 5600 MT/s 和时序不属于这一章的继续提高项目。

### 02：已经安装的软件

把原 06 提升为 02：

- MCHOSE HUB；
- MSI Afterburner；
- HWiNFO 仅传感器模式；
- AIDA64、TM5、ZenTimings 先询问 AI 再使用。

重点说明每个工具“什么时候用”，不扩写成软件百科。

### 03：当前关键设置与截图

后置展示 CPU、功耗、显卡和 UXTU 的当前设置证据：

- Windows 电源计划 4500/5000MHz；
- SPL 65W、sPPT 85W、fPPT 85W；
- 独显直连；
- 控制台高性能电源模式关闭；
- UXTU All Core Offset -20。

这些内容作为首屏摘要的复现证据，不再另做第二套基线卡片。

### 04：异常恢复与教程

集中放置：

- UXTU 无法启动时删除配置目录的恢复步骤与截图；
- 两条 B 站教程的完整标题、BV 号、用途、时间点和链接；
- BIOS/UMAF 的进入方式只在需要恢复时说明。

教程仍是辅助资料，不再打断“下一步”和“工具”主线。

### 页尾附录：内存超频与时序记录

该附录始终直接可见，不使用折叠、手风琴或弹窗。

保留：

- 16GB × 2 双通道；
- 当前稳定 5600 MT/s；
- “内存超频和时序已经调整完成”的明确说明；
- ZenTimings 当前稳定结果截图；
- DDR SPD Timing 与 DDR Non-SPD Timing 两张 UMAF 截图；
- 约 80MB UMAF 分区不可删除的警告；
- 只按已记录字段回退、未记录参数不得猜测的说明。

附录的目的为复现和回退，不把时序记录包装成鼓励继续超频的教程。

## 5. 去重规则

- 当前状态只在首屏集中总结一次；
- 后置章节只解释操作位置、证据图和恢复路径，不再复制整套摘要；
- 允许截图标题重复必要的参数值，但不得再次出现与首屏相同的大型基线网格；
- 页尾内存附录可以重述 5600 MT/s 与时序已调整，因为它承担详细证据与回退职责，且与首屏相隔完整正文。

## 6. 图片映射

10 张图片全部保留：

- 设备图：首屏；
- CPU 频率、CPU 功耗、显卡模式、控制台电源模式、UXTU 负压：章节 03；
- UXTU 恢复：章节 04；
- ZenTimings、DDR SPD Timing、DDR Non-SPD Timing：页尾附录。

图片保持原始比例和当前无损质量，不新增点击放大或折叠交互。

## 7. 测试与验收

内容测试应验证：

- 首屏源码顺序中明确出现“内存超频”“时序已调整”“5600 MT/s”“16GB × 2”；
- 不再存在独立的 `00 当前设置基线`；
- 章节编号和顺序为 01“之后可以怎么调”、02“已经安装的软件”，然后才出现设置截图和教程；
- 10 张图片仍全部映射且只出现一次；
- 内存/UMAF 附录位于教程和设置截图之后；
- UMAF 警告与回退说明仍存在；
- 页面不出现折叠控件或需要点击才能显示的证据图。

构建后执行完整 HTML validator、桌面与移动端浏览器检查，确认首屏信息密度、章节顺序、全页滚动和图片比例正常。

## 8. 范围

不删除用户提供的内存超频、时序、教程或设置证据；不改变任何记录值；不增加新调校建议；不改图片质量和交互原则。
